# Ansys

3Studio 可以连接你电脑上已安装的 **Ansys**，让 Agent 直接替你做两类仿真：

- **结构仿真**（基于 MAPDL，Ansys 的结构求解器）：搭建结构模型、划分网格、施加约束和载荷，做模态分析（算固有频率和振型）或静力分析（算位移和应力）。
- **电磁仿真**（基于 HFSS，运行在 AEDT——Ansys Electronics Desktop 里）：天线、微波器件这类高频电磁场仿真，求解后读取 S 参数。

> ⚠️ **仅支持 Windows，且需要有效的 Ansys 软件许可证。** 使用前提、后台桥接程序的工作方式、通用 Python 环境设置和常见连接问题，见《[概览与通用设置](./cae-overview.md)》——本页只写 Ansys 特有的部分。

## 配置 Ansys

打开**设置 → 工具**标签页，往下滚动能看到 **Ansys** 卡片。卡片默认折叠，只显示标题和状态徽章；点击标题（或右侧箭头）展开后才能看到下面全部设置。

**状态徽章**：卡片标题右侧的徽章常见有"已检测到"（绿色）、"未检测到"、"需准备环境"（金色，表示已探测到安装但对应的 Python 仿真环境还没准备好）、"准备中…"（金色，环境正在准备，旁边会显示当前阶段文案）、"出错"（红色）、"就绪"（绿色，引擎正在运行）。点击"检测"刷新后，徽章下方的版本行会显示成「MAPDL x · AEDT y」这样的组合格式（两个产品都装了的话）；只装了一个则只显示这一个的版本号。

**检测**：点击"检测"按钮，重新探测 MAPDL / AEDT 的版本与安装路径，不需要你手动选路径。

**准备环境（首次使用必做）**：探测到 Ansys 安装后，卡片状态会先变成"需准备环境"，这时会出现一个"准备环境"按钮。点击后 3Studio 会在后台下载并搭建 Ansys 桥接程序依赖的 Python 仿真环境，按钮下方会提示"需要下载 Python 仿真环境（约 1.2 GB，首次约 30 分钟）"——请确保联网并耐心等待，准备好之后状态会变成"就绪"。准备过程中会依次显示进度阶段（解析解释器 / 创建虚拟环境 / 安装依赖 / 就绪探测），旁边有一个"取消"按钮可以随时中止。

**Python 解释器（手动指定）**：Ansys 桥接需要 Python 3.10–3.12。不填的话会用自动兼容逻辑；也可以点"选择 Python…"手动指定一个解释器路径。更换后不用立即重启整个应用，会在该引擎**下次启动或重建环境时**生效。

**依赖来源**：一组单选项，决定"准备环境"时依赖从哪里装：

- **自动（公共 PyPI / 镜像）**：默认选项，联网自动安装。
- **使用我已有的 Python 环境**：选中后出现"选择…"按钮，指定一个你自己已经装好所需依赖的解释器；下次点"准备环境"会校验这个解释器。

**环境位置**：Ansys 的隔离 Python 环境默认建在应用数据目录下，卡片上有"环境位置"一行显示当前位置，点"更改位置…"可以换到别的磁盘（比如系统盘空间紧张时）；更改后下次"准备环境"会在新位置构建，旧目录不会自动删除。

**重建隔离环境**：这个按钮随时都在，点一下会用当前的基座 Python 重新搭建 Ansys 的隔离环境——比如换了通用基座 Python 之后想让 Ansys 立即用上新版本，就点它。

**通用基座回退提示**：如果设置页顶部通用"Python 环境"卡片里指定的基座 Python 不兼容 Ansys（不在 3.10–3.12 范围内），卡片上会出现一行金色提示，说明 Ansys 已自动回退到自己的发现逻辑，不影响正常使用；完整说明见《[概览与通用设置](./cae-overview.md)》。

**桥接端口**：只读显示，正常情况下不需要改。

**测试连接**：点击"测试连接"按钮，3Studio 会尝试跟 Ansys 桥接程序握手，成功时显示：`Status`（桥接进程状态）、`Sidecar`（若有）、`Jobs active`（当前活跃任务数，若有）、`Ansys detected`（是否检测到 Ansys 安装，Yes / No）。

**MAPDL / AEDT 检测行**：位于卡片展开后的最下方（"测试连接"按钮之上），3Studio 会分别检测两类 Ansys 安装：

- **Ansys 统一安装（MAPDL）**：通过 `AWP_ROOT<版本号>` 环境变量检测（Ansys 统一安装器写入）。
- **Ansys EM（HFSS / AEDT）**：通过 `ANSYSEM_ROOT<版本号>` 环境变量检测（AEDT 独立安装器写入）。

这两类产品各占一行，显示检测状态（"已检测（环境变量）"／"手动指定"／"未找到"）和识别到的版本号。任一类被检测到即可启用 Ansys 引擎——如果你的机器只安装了 AEDT（没有装 MAPDL），HFSS 相关功能同样可以正常使用。

如果环境变量没有设置，或者安装位置不是标准路径，可以点对应行的"选择安装目录…"手动指定：

- MAPDL：选择版本目录（例如 `C:\Program Files\ANSYS Inc\v251`）。
- AEDT：选择版本目录（例如 `C:\Program Files\AnsysEM\v242`）或其 `Win64` 子目录。

手动指定的目录优先于环境变量检测；点"清除"恢复自动检测。如果指定时 Ansys 引擎已经在运行，需要重启应用才能生效，卡片下方会出现提示。

## 连接状态徽章

侧边栏底部的"运行时"面板（点侧边栏底部的"运行时"按钮展开）里有一项 **Ansys**，显示 Ansys 桥接后台进程当前的运行状态：运行中 / 已停止 / 错误 / 不可用。跟 MATLAB 的 Bridge 徽章不同，这几个状态文案会跟随界面语言切换。

## 让 Agent 做结构仿真（MAPDL）

配置好之后，你可以在对话里直接让 Agent 做结构分析，Agent 会按"建模 → 网格 → 约束 → 求解 → 读结果"的顺序自动组织这些步骤：

- **搭建结构模型**：目前支持块体（长方体）几何，指定尺寸、材料参数（弹性模量、泊松比、密度）和单元类型。建好后长方体的六个面会自动获得名字（比如 FACE_ZMIN 表示 Z 方向最小的那个面），后续施加约束和载荷时 Agent 直接按名字引用某个面。
- **划分网格**：按指定的目标单元边长把模型划分成网格。
- **施加约束**：把某些面完全固定住，或者只约束某些方向的自由度。
- **求解**：两种分析类型——**模态分析**（提取前 N 阶固有频率，默认 6 阶）和**静力分析**（可施加整体加速度、面上的力、面压强等载荷）。求解在后台任务里执行，不会卡住对话，进度会实时显示；每次分析的原生结果文件（`.rst` / `.db`）会保存到工作区的一个子文件夹里，方便你之后拿到 Ansys 里做进一步后处理。
- **读取结果**：模态分析返回各阶固有频率、振型（每阶的最大位移及其位置）和有效质量（含累计百分比——Agent 会据此判断提取的阶数够不够，不够会自动增加）；静力分析返回位移、von Mises 等效应力和支反力。
- **查询局部结果**：不止看整体最大值——Agent 可以对某个命名面、某些节点、甚至某个空间坐标位置，查询位移、应力、应变等具体分量，并做最大值定位这类统计。比如"根部截面的最大弯曲应力出现在哪里"这种问题可以直接问。
- **执行底层 APDL 命令**：以上步骤覆盖不了的特殊几何、材料或分析类型，Agent 可以直接执行 Ansys 原生的 APDL 命令。这是一个直达底层、影响面较大的操作，默认会先弹出审批卡片等你确认（审批卡片的用法见《[与 Agent 对话](./chat.md)》）；同时出于安全考虑，调用系统命令、读写工作区之外文件的命令会被直接拦截。

## 让 Agent 做电磁仿真（HFSS）

HFSS 相关能力需要机器上装有 AEDT（检测行里"Ansys EM"显示已检测）：

- **启动 / 关闭 HFSS 会话**：首次使用时 Agent 会先启动 AEDT（后台任务，冷启动可能需要几分钟）；用完之后可以让 Agent 关闭会话，释放桌面程序和软件许可证。
- **建模与设置**：画几何、设边界条件、配材料、建求解设置（Setup）、做后处理，这些都通过 Agent 执行脚本完成。执行脚本属于影响面较大的操作，默认会先弹出审批卡片等你确认。
- **求解**：对指定的 Setup 求解，同样在后台任务里执行（默认用 4 个求解核心，可以让 Agent 调整）；求解失败时 Agent 能拿到求解日志的末尾内容，自行诊断原因。
- **读取 S 参数**：求解完成后按端口读取 S 参数矩阵，支持 dB、幅值、相位、实部、虚部几种形式；返回结果时会附带收敛信息，Agent 会据此判断结果可信度。
