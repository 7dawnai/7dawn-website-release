# MATLAB 与 STK 集成

3Studio 可以连接你电脑上已安装的 **MATLAB**（含 Simulink）和 **STK**（Systems Tool Kit，卫星轨道与任务分析软件），让 Agent 直接替你执行脚本、建模、仿真、轨道计算这些工作，而不用你自己在这些软件里手动点。

> ⚠️ **仅支持 Windows。** MATLAB 和 STK 集成功能只在 Windows 系统上可用；在 macOS 或 Linux 上，设置页面里不会出现这两个软件的配置区，Agent 也不具备对应能力。

用之前还需要满足两个前提：

- 你的电脑上已经**装好并能正常打开** MATLAB / STK 本身——3Studio 不包含也不安装这两款软件，它只是去连接已经装好的它们。
- 3Studio 会在后台启动一个桥接程序（Bridge）来跟 MATLAB / STK 通信，这个桥接程序依赖一个本地 Python 环境，通常首次使用时会自动准备好，也可以在设置里手动指定。

## Python 环境（通用设置）

打开**设置 → 工具**标签页，最上方（MATLAB / STK 卡片之上）有一张通用的 **Python 环境** 卡片，用来统一管理这些桥接程序背后依赖的 Python 环境。默认折叠，点标题展开后有两行设置：

- **基座 Python**：默认「自动发现」（3Studio 自动在系统里找一个可用的解释器）；也可以点「选择 python.exe…」手动指定一个版本。指定之后下方会列出各引擎对这个版本的兼容情况——「兼容」或「不兼容（该引擎将回退自动发现）」；想恢复自动发现，点「恢复自动」即可。
- **虚拟环境根目录**：各引擎会在这个目录下各自创建独立的隔离环境（比如 STK 用 `stk-venv`）。默认是应用数据目录，可以点「更改位置…」换到别的磁盘（比如系统盘空间紧张时），点「恢复默认」还原。

> **MATLAB 不受这张卡片影响**：MATLAB 使用它自带的 Python，不参与这里的基座 / 目录设置，也不会出现在兼容性列表里。

改动不会立即生效，而是在各引擎**下次启动或重建环境时**才应用；已经就绪的引擎不受影响，如果想立即切换，展开下面对应的引擎卡片（比如 STK），点「重建隔离环境」按钮——这个按钮随时都在，不需要等出现任何提示。

## 配置 MATLAB

打开**设置 → 工具**标签页，往下滚动能看到 **MATLAB** 卡片。卡片默认折叠，只显示标题和状态标签；点击标题（或右侧箭头）展开后才能看到下面的检测 / Python 解释器 / 测试连接等设置。

- **状态标签**：卡片顶部有一个状态标签，常见的有"已检测到"（绿色，路径已设置且能识别出版本）、"未检测到"（还没设置有效路径）、"出错"（红色，准备环境时失败，下方会显示具体原因，并出现一个"重试"按钮）。
- **选择 MATLAB 安装目录**：点击"选择 MATLAB 文件夹…"，在弹出的文件夹选择器里选中你电脑上 MATLAB 的安装目录。选中后卡片会显示识别到的版本号和路径；如果选的目录不对，会提示"无效的 MATLAB 路径"。
- **Python 解释器**：桥接程序需要 Python 3.9–3.11。如果不填，会用系统默认的兼容版本；也可以点"选择 Python…"手动指定一个解释器路径。**更换解释器后需要重启应用才生效**。
- **桥接端口**：只读显示，正常情况下不需要改。
- **测试连接**：点击"测试连接"按钮，3Studio 会尝试跟桥接程序握手，成功时会显示几项信息：`Status`（桥接进程状态）、`MATLAB Engine`（MATLAB 引擎是否成功启动）、`Workspace`（当前使用的工作目录）；如果 MATLAB 引擎启动失败，还会额外显示 `engine_error` 的具体错误信息。

### MATLAB 的连接状态徽章

除了设置页面里的卡片，侧边栏底部还有一个独立的"运行时"按钮，点开会展开一个小面板，里面有一项 **Bridge**，专门显示 MATLAB 桥接后台进程当前的运行状态：

| 显示文案 | 含义 |
|---|---|
| Running | 桥接进程正常运行 |
| Error | 桥接进程出错 |
| Stopped | 桥接进程已停止 |
| Unavailable | 桥接进程还未启动 / 暂不可用 |

> 这几个状态文案目前是英文原文，不随界面语言切换。

## 配置 STK

同样在**设置 → 工具**标签页里，往下滚动能看到 **STK** 卡片，同样默认折叠，点击标题（或右侧箭头）展开后才能看到全部设置。跟 MATLAB 相比，主要差异是：

- **有"检测"按钮**：点击"检测"，3Studio 会自动探测你电脑上是否装了 STK（不需要你手动选路径）；检测到时会显示版本号，检测不到时会在下方给出原因提示。
- **Python 解释器（手动指定）**：STK 桥接需要 Python 3.9–3.13。手动指定的解释器现在是最高优先级的覆盖项；不指定时依次使用通用基座 Python（设置页顶部的「Python 环境」卡片，见上一节）、自动发现。点「选择 Python…」手动指定；更换后不用重启整个应用，会在该引擎**下次启动或重建环境时**生效。
- **隔离环境位置**：STK 的 Python 依赖默认安装在一个独立的虚拟环境里（跟你系统上的 Python 互不影响），卡片上有"环境位置"一行显示当前位置，可以点"更改位置…"换到别的磁盘（比如 C 盘空间不够时）。展开后的卡片上一直有一个"重建隔离环境"按钮，点一下就会用当前的基座 Python 重新搭建这个隔离环境（比如切换了通用基座之后想让 STK 立即用上新版本，就点它）。如果检测到当前用的其实是系统 Python 而不是这个隔离环境，还会额外提示"使用系统 Python（未隔离）"，提醒你去点这个按钮修复。
- **测试连接**：跟 MATLAB 类似，点击后显示 `Status`（桥接进程状态）、`Sidecar`（STK 侧的后台服务状态，若有）、`STK`（STK 软件本身的连接状态，若有）；连接失败时会显示 `stk_error` 的具体信息。

> **STK 没有像 MATLAB 那样的侧边栏"运行时"连接状态徽章**——STK 的连接状态只能在设置页面的这张卡片里查看，不会出现在侧边栏底部的"运行时"面板中。

## 让 Agent 使用 MATLAB

配置好之后，你可以在对话里直接让 Agent 帮你做以下这些事，Agent 会自动选用对应的能力，不需要你了解具体是怎么调用的：

- **执行 MATLAB 脚本 / 命令**：比如让 Agent 帮你算一段公式、跑一段 `.m` 脚本，变量会在同一次对话里持续保留，后续可以接着引用。
- **运行 Simulink 仿真**：Agent 可以直接跑一个已有的 Simulink 模型，拿到关键结果摘要、自动生成的图表，并保留完整的仿真数据文件供后续深入分析。
- **搭建 / 修改 Simulink 模型**：新建模型、往模型里添加功能模块（比如积分器、增益模块）、批量修改模型结构（一次性加模块、连线、删除、改参数），以及在模型卡住时强制重置。
- **读取仿真结果**：从 MATLAB 工作区里取回某个变量的值，或者把之前仿真产生的数据文件重新加载进来做进一步计算。
- **导出图表**：把时间序列数据画成图并导出为 PNG 或 SVG 图片文件，方便你在报告里直接使用。
- **在 MATLAB 编辑器里打开文件**：让 Agent 帮你在 MATLAB 里打开某个模型或脚本文件。这个操作会执行文件自带的加载脚本，属于有风险的操作——默认情况下会先弹出一张审批卡片等你确认（审批卡片的用法见《[与 Agent 对话](./chat.md)》）。

## 让 Agent 使用 STK

配置好 STK 之后，Agent 具备的能力围绕"搭场景、算轨道"展开：

- **创建仿真场景**：新建一个 STK 场景，可以指定起止时间；首次调用会自动启动 STK 桌面程序（需要十几秒）。
- **添加卫星**：按开普勒轨道根数（近地点/远地点高度、倾角、升交点赤经等）把一颗卫星加进当前场景，并可以指定轨道传播器类型（如 J4 摄动模型、HPOP 高精度模型等）。
- **添加地面站**：按经纬度和海拔高度，把一个地面设施（比如地面站）加进当前场景。
- **重新计算轨道**：对已有卫星重新执行一次轨道传播计算。
- **计算可见性窗口**：计算两个对象（比如某颗卫星和某个地面站）之间什么时候能"看得见"彼此，返回具体的时间区间和总时长。
- **执行底层 STK 命令**：对于以上几类做不到的更细操作（比如设置卫星姿态、控制场景动画播放、调整三维显示），Agent 也可以直接执行一条 STK 原生命令。这是一个直达底层、影响面较大的操作，同样默认会先弹出审批卡片等你确认。

## 常见连接问题

以下问题目前能在代码和设置界面里找到依据，遇到其他情况建议直接把测试连接返回的错误信息发给 Agent 帮你诊断。

- **MATLAB 卡片显示"未检测到"**：还没有选择 MATLAB 安装目录，去"选择 MATLAB 文件夹…"里重新选一次。
- **提示"无效的 MATLAB 路径"**：选中的文件夹不是一个有效的 MATLAB 安装目录，检查一下路径是否选对了。
- **MATLAB 换了 Python 解释器后好像没生效**：这是预期行为——MATLAB 更换解释器必须**重启应用**才会生效。
- **STK 换了「Python 解释器（手动指定）」后好像没生效**：这也是预期行为，但触发方式不同——手动指定的解释器会在该引擎**下次启动或重建环境时**生效；想立即生效，展开 STK 卡片直接点「重建隔离环境」即可，不用重启整个应用。
- **引擎卡片显示"通用基座 Python 不兼容…已回退自动发现"是什么意思**：说明你在通用「Python 环境」卡片里指定的基座 Python 版本，不在这个引擎的支持范围内，该引擎已经自动回退到自己的自动发现逻辑，不影响正常使用。如果想让这个引擎也统一用你指定的基座，换一个各引擎都兼容的版本即可（比如 3.11）。
- **卡片显示"出错"**：下方会附带具体的错误说明；可以直接点"重试"按钮让 3Studio 重新准备一次。
- **"测试连接"按钮点了但显示失败**：说明桥接程序当前连不上，先确认 MATLAB / STK 软件本身能正常打开，再确认路径 / 解释器配置无误后重新测试。
- **STK 卡片提示"使用系统 Python（未隔离）"**：说明当前用的解释器不是 STK 专属的隔离环境，展开卡片点"重建隔离环境"让 3Studio 重新搭建一个独立环境（这个按钮平时也一直在，不是只有出现这条提示才有）。

> ⚠️ 待核对：桥接进程处于 Error / Stopped 状态时，除了"重试"按钮外是否还有其他官方推荐的恢复步骤（例如是否需要重启应用），当前代码里没有找到明确说明，建议后续与研发确认后补充。
