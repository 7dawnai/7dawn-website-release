# MATLAB 与 Simulink

3Studio 可以连接你电脑上已安装的 **MATLAB**（含 Simulink），让 Agent 直接替你执行脚本、建模、仿真这些工作。

> ⚠️ **仅支持 Windows。** 使用前提、后台桥接程序的工作方式、通用 Python 环境设置和常见连接问题，见《[概览与通用设置](./cae-overview.md)》——本页只写 MATLAB 特有的部分。

## 配置 MATLAB

打开**设置 → 工具**标签页，往下滚动能看到 **MATLAB** 卡片。卡片默认折叠，只显示标题和状态标签；点击标题（或右侧箭头）展开后才能看到下面的检测 / Python 解释器 / 测试连接等设置。

- **状态标签**：卡片顶部有一个状态标签，常见的有"已检测到"（绿色，路径已设置且能识别出版本）、"未检测到"（还没设置有效路径）、"出错"（红色，准备环境时失败，下方会显示具体原因，并出现一个"重试"按钮）。
- **选择 MATLAB 安装目录**：点击"选择 MATLAB 文件夹…"，在弹出的文件夹选择器里选中你电脑上 MATLAB 的安装目录。选中后卡片会显示识别到的版本号和路径；如果选的目录不对，会提示"无效的 MATLAB 路径"。
- **Python 解释器**：桥接程序需要 Python 3.9–3.11。如果不填，会用系统默认的兼容版本；也可以点"选择 Python…"手动指定一个解释器路径。**更换解释器后需要重启应用才生效**。
- **桥接端口**：只读显示，正常情况下不需要改。
- **测试连接**：点击"测试连接"按钮，3Studio 会尝试跟桥接程序握手，成功时会显示几项信息：`Status`（桥接进程状态）、`MATLAB Engine`（MATLAB 引擎是否成功启动）、`Workspace`（当前使用的工作目录）；如果 MATLAB 引擎启动失败，还会额外显示 `engine_error` 的具体错误信息。

> **MATLAB 不受通用「Python 环境」卡片影响**：MATLAB 使用它自带的 Python，不参与设置页顶部那张卡片的基座 / 目录设置。

## 连接状态徽章

除了设置页面里的卡片，侧边栏底部还有一个独立的"运行时"按钮，点开会展开一个小面板，里面有一项 **Bridge**，专门显示 MATLAB 桥接后台进程当前的运行状态：

| 显示文案 | 含义 |
|---|---|
| Running | 桥接进程正常运行 |
| Error | 桥接进程出错 |
| Stopped | 桥接进程已停止 |
| Unavailable | 桥接进程还未启动 / 暂不可用 |

> 这几个状态文案目前是英文原文，不随界面语言切换。

## 让 Agent 使用 MATLAB

配置好之后，你可以在对话里直接让 Agent 帮你做以下这些事，Agent 会自动选用对应的能力，不需要你了解具体是怎么调用的：

- **执行 MATLAB 脚本 / 命令**：比如让 Agent 帮你算一段公式、跑一段 `.m` 脚本，变量会在同一次对话里持续保留，后续可以接着引用。
- **运行 Simulink 仿真**：Agent 可以直接跑一个已有的 Simulink 模型，拿到关键结果摘要、自动生成的图表，并保留完整的仿真数据文件供后续深入分析。
- **搭建 / 修改 Simulink 模型**：新建模型、往模型里添加功能模块（比如积分器、增益模块）、批量修改模型结构（一次性加模块、连线、删除、改参数），以及在模型卡住时强制重置。
- **读取仿真结果**：从 MATLAB 工作区里取回某个变量的值，或者把之前仿真产生的数据文件重新加载进来做进一步计算。
- **导出图表**：把时间序列数据画成图并导出为 PNG 或 SVG 图片文件，方便你在报告里直接使用。
- **在 MATLAB 编辑器里打开文件**：让 Agent 帮你在 MATLAB 里打开某个模型或脚本文件。这个操作会执行文件自带的加载脚本，属于有风险的操作——默认情况下会先弹出一张审批卡片等你确认（审批卡片的用法见《[与 Agent 对话](./chat.md)》）。
