# STK

3Studio 可以连接你电脑上已安装的 **STK**（Systems Tool Kit，卫星轨道与任务分析软件），让 Agent 直接替你搭场景、算轨道，而不用你自己在 STK 里手动点。

> ⚠️ **仅支持 Windows。** 使用前提、后台桥接程序的工作方式、通用 Python 环境设置和常见连接问题，见《[概览与通用设置](./cae-overview.md)》——本页只写 STK 特有的部分。

## 配置 STK

打开**设置 → 工具**标签页，往下滚动能看到 **STK** 卡片，默认折叠，点击标题（或右侧箭头）展开后才能看到全部设置。跟 MATLAB 相比，主要差异是：

- **有"检测"按钮**：点击"检测"，3Studio 会自动探测你电脑上是否装了 STK（不需要你手动选路径）；检测到时会显示版本号，检测不到时会在下方给出原因提示。
- **Python 解释器（手动指定）**：STK 桥接需要 Python 3.9–3.13。手动指定的解释器是最高优先级的覆盖项；不指定时依次使用通用基座 Python（设置页顶部的「Python 环境」卡片，见《[概览与通用设置](./cae-overview.md)》）、自动发现。点「选择 Python…」手动指定；更换后不用重启整个应用，会在该引擎**下次启动或重建环境时**生效。
- **隔离环境位置**：STK 的 Python 依赖默认安装在一个独立的虚拟环境里（跟你系统上的 Python 互不影响），卡片上有"环境位置"一行显示当前位置，可以点"更改位置…"换到别的磁盘（比如 C 盘空间不够时）。展开后的卡片上一直有一个"重建隔离环境"按钮，点一下就会用当前的基座 Python 重新搭建这个隔离环境（比如切换了通用基座之后想让 STK 立即用上新版本，就点它）。如果检测到当前用的其实是系统 Python 而不是这个隔离环境，还会额外提示"使用系统 Python（未隔离）"，提醒你去点这个按钮修复。
- **测试连接**：跟 MATLAB 类似，点击后显示 `Status`（桥接进程状态）、`Sidecar`（STK 侧的后台服务状态，若有）、`STK`（STK 软件本身的连接状态，若有）；连接失败时会显示 `stk_error` 的具体信息。

> **STK 没有侧边栏"运行时"面板里的连接状态徽章**——STK 的连接状态只能在设置页面的这张卡片里查看，不会出现在侧边栏底部的"运行时"面板中。

## 让 Agent 使用 STK

配置好 STK 之后，Agent 具备的能力围绕"搭场景、算轨道"展开：

- **创建仿真场景**：新建一个 STK 场景，可以指定起止时间；首次调用会自动启动 STK 桌面程序（需要十几秒）。
- **添加卫星**：按开普勒轨道根数（近地点/远地点高度、倾角、升交点赤经等）把一颗卫星加进当前场景，并可以指定轨道传播器类型（如 J4 摄动模型、HPOP 高精度模型等）。
- **添加地面站**：按经纬度和海拔高度，把一个地面设施（比如地面站）加进当前场景。
- **重新计算轨道**：对已有卫星重新执行一次轨道传播计算。
- **计算可见性窗口**：计算两个对象（比如某颗卫星和某个地面站）之间什么时候能"看得见"彼此，返回具体的时间区间和总时长。
- **执行底层 STK 命令**：对于以上几类做不到的更细操作（比如设置卫星姿态、控制场景动画播放、调整三维显示），Agent 也可以直接执行一条 STK 原生命令。这是一个直达底层、影响面较大的操作，默认会先弹出审批卡片等你确认（审批卡片的用法见《[与 Agent 对话](./chat.md)》）。
