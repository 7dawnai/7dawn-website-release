# MBSE 系统建模

MBSE（Model-Based Systems Engineering，基于模型的系统工程）是一种用"模型"而不是一摞文档来描述系统需求、结构和行为的工程方法。3Studio 内置了对 **SysML v2** 建模语言的支持：Agent 负责替你编写模型文件，你在专门的 **MBSE 工作台**里以树、表格、矩阵、图形四种方式浏览和审查模型。

> ⚠️ **实验性功能。** MBSE 建模能力目前处于实验阶段，界面和能力可能在后续版本中调整。

这个功能的分工方式和传统建模软件（如 Cameo、MagicDraw）不同——**你不在图上直接画模型**。模型的"源头"是工作区里的文本文件，由 Agent 编写和修改；工作台里看到的树、表、矩阵、图都是从这些文件自动生成的**只读视图**。想改模型，就在对话里告诉 Agent 要改什么。

## 准备工作

- **模型文件**：SysML v2 模型是纯文本文件，扩展名为 `.sysml`（配套的 KerML 文件为 `.kerml`），放在你的工作区（你打开的项目文件夹）里即可被识别。
- **MBSE 建模师 Agent**（推荐）：新建 Agent 时（Sidebar 的"新建 Agent"，见《[Agent 与会话管理](./agents.md)》），模板列表里选择"**MBSE 建模师**"（📐）。这个内置模板专门做 SysML v2 建模，自带"写模型 → 校验 → 按报错修正"的工作习惯；用其他 Agent 也可以，但建模质量和效率通常不如它。

## 打开模型：两级入口

**第一级：文件视图。** 当 Agent 在 Computer 面板（右侧展示 Agent 操作过程的面板）里打开一个 `.sysml` 或 `.kerml` 文件时，会自动以"模型视图"展示——顶部有一个 **Model / 源码** 切换开关：Model 模式显示解析后的模型结构，源码模式显示原始文本。

**第二级：MBSE 工作台。** 文件视图顶栏还有一个 **⟂ MBSE** 按钮，点击后进入全屏的 MBSE 工作台——这里加载的不是单个文件，而是**整个工作区**里所有的 SysML 模型文件，适合总览和审查。工作台左上角的 **◂ Back** 按钮可随时返回主界面。

> 首次进入工作台时会显示"parsing workspace…"的加载提示——第一次需要加载 SysML 标准库，通常要等数秒到十几秒，之后会快很多。

## 工作台布局

工作台从左到右分为几个可拖拽调宽的面板：

| 面板 | 内容 |
|---|---|
| 树 | 整个工作区模型的结构大纲，顶部带搜索框 |
| 模型（中央） | 当前"投影"的主视图（见下节），底部是问题诊断条 |
| 细节 | 选中元素的属性与关联（仅 Diagram 投影时显示为独立面板） |
| chat | 内嵌的 Agent 对话区，顶部可切换 agent 和 session |

树、细节、chat 三个侧面板都可以点 ◂ / ▸ 折叠成一条细边，给中央视图腾地方；再点一次展开。

在树、表格、矩阵里点击任何一行都会选中该元素，各视图会同步定位。顶栏的 **← / →** 按钮是浏览历史的"上一步 / 下一步"，看深了随时退回来。

## 四种投影

顶栏的切换开关可以在四种"投影"（同一个模型的不同呈现方式）之间切换：

- **Model**：选中元素的详情页——类型、限定名（元素在模型里的完整路径名）、源码片段，以及它的上级、子元素和关系。
- **Table**：把整个工作区的模型元素拍平成一张可排序的表格（列：Name / Kind / Owner / Typing / Mult / Value），适合批量核对。
- **Matrix**：可追溯性矩阵，内置 5 个预设——**Satisfy**（需求满足）、**Verify**（需求验证）、**Derive**（需求派生）、**Refine**（需求细化）、**N²**（接口关系矩阵）。右上角显示覆盖率，点行列表头可跳转到对应元素。
- **Diagram**：图形视图，支持 4 种图，用子开关切换——**BDD**（块定义图）、**IBD**（内部块图）、**STM**（状态机图）、**ACT**（活动图）。

> 四种投影都是**只读**的——它们是模型文件的自动呈现，不能在上面直接编辑。

### Diagram 视图的操作

- **双击节点**：下钻，以该元素为根重新展开图。
- **右键节点**：定位到树 / 下钻到此 / 打开源码 / 带入 chat / 复制限定名。
- **右键连线**：跳到源端 / 跳到靶端 / 带入 chat。
- **右键空白处**：适配视图（缩放到刚好看全）/ 回到根（退出下钻）。

"打开源码"会弹出一个只读的源码预览浮层，直接定位到该元素在文件里的声明行，看完按关闭即可——它只是预览，不是编辑器。

## 与 Agent 协作改模型

工作台的核心用法是"你发现问题 → 交给 Agent 改"，为此内置了三条快捷通路：

- **诊断一键修**：中央面板底部的问题诊断条列出模型里的语法和引用错误，每条右侧有一个"**修 ▸**"按钮，点击就把这条报错（连同出错的源码行）直接发给 chat 面板里的 Agent，让它去修。（需要 chat 面板里已有会话；没有时按钮会提示先新建。）
- **带入 chat**：在细节面板点"**@ 带入 chat**"，或在图上右键选"带入 chat"，会把该元素作为一个引用标签插入 chat 输入框——接着描述你的要求即可，比如"把这个部件的多重性改成 1..4"。
- **自动刷新**：Agent 修改了模型文件后，工作台会自动重新解析并刷新所有视图，你不需要手动做任何操作。如果某次修改导致解析失败，工作台会保留上一次成功的模型内容，并在顶栏显示"parse failed"标记——把报错交给 Agent 修好后即恢复。

## 注意事项

- **首次加载慢是正常的**：SysML 标准库第一次加载约需数秒到十几秒，之后有缓存。
- **大模型也能用**：树和表格针对上千元素的模型做过性能优化，可以正常浏览；但图形视图一次展示的内容取决于当前下钻位置，看不清时先下钻到关注的子系统。
- **部分界面文案为英文**：投影名（Model / Table / Matrix / Diagram）、图类型（BDD / IBD / STM / ACT）、矩阵预设名、表格列头等专业术语标签目前为英文原文，不随界面语言切换。
- **Agent 的校验不是完整仿真**：MBSE 建模师用的校验工具检查的是语法和引用正确性，不做完整的语义证明或数值仿真；需要仿真时结合 MATLAB 等工具（见《[MATLAB 与 STK 集成](./matlab-stk.md)》）。
