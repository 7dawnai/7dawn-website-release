# 支持的 CAE/CAD 软件：概览与通用设置

3Studio 可以连接你电脑上已经安装的专业工程软件，让 Agent 直接替你执行脚本、建模、仿真、计算这些工作，而不用你自己在这些软件里手动点。当前支持三款：

| 软件 | 能做什么 | 详细说明 |
|---|---|---|
| **MATLAB**（含 Simulink） | 执行脚本与公式计算、搭建和运行 Simulink 仿真、导出图表 | 《[MATLAB 与 Simulink](./matlab.md)》 |
| **STK**（Systems Tool Kit） | 卫星轨道与任务分析：搭场景、加卫星和地面站、算可见性窗口 | 《[STK](./stk.md)》 |
| **Ansys** | 结构仿真（模态 / 静力分析）与电磁仿真（HFSS） | 《[Ansys](./ansys.md)》 |

> ⚠️ **仅支持 Windows。** 这三款软件的集成功能只在 Windows 系统上可用；在 macOS 或 Linux 上，设置页面里不会出现它们的配置区，Agent 也不具备对应能力。

## 使用前提

- 你的电脑上已经**装好并能正常打开**对应软件本身——3Studio 不包含也不安装这些软件，它只是去连接已经装好的它们（Ansys 还需要有效的软件许可证）。
- 3Studio 会在后台启动一个**桥接程序（Bridge）**来跟这些软件通信，桥接程序依赖一个本地 Python 环境，通常首次使用时会自动准备好，也可以在设置里手动指定（见下一节）。

## Python 环境（通用设置）

打开**设置 → 工具**标签页，各软件卡片之上有一张通用的 **Python 环境** 卡片，用来统一管理这些桥接程序背后依赖的 Python 环境。默认折叠，点标题展开后有两行设置：

- **基座 Python**：默认「自动发现」（3Studio 自动在系统里找一个可用的解释器）；也可以点「选择 python.exe…」手动指定一个版本。指定之后下方会列出各引擎对这个版本的兼容情况——「兼容」或「不兼容（该引擎将回退自动发现）」；想恢复自动发现，点「恢复自动」即可。
- **虚拟环境根目录**：各引擎会在这个目录下各自创建独立的隔离环境（比如 STK 用 `stk-venv`）。默认是应用数据目录，可以点「更改位置…」换到别的磁盘（比如系统盘空间紧张时），点「恢复默认」还原。

> **MATLAB 不受这张卡片影响**：MATLAB 使用它自带的 Python，不参与这里的基座 / 目录设置，也不会出现在兼容性列表里。

改动不会立即生效，而是在各引擎**下次启动或重建环境时**才应用；已经就绪的引擎不受影响。如果想立即切换，展开下面对应的引擎卡片（比如 STK 或 Ansys），点「重建隔离环境」按钮——这个按钮随时都在，不需要等出现任何提示。

各引擎对 Python 版本的要求不同（MATLAB 3.9–3.11、STK 3.9–3.13、Ansys 3.10–3.12），想让所有引擎统一用你指定的基座，选一个都兼容的版本即可（比如 3.11）。

## 侧边栏的「运行时」面板

侧边栏底部有一个独立的「运行时」按钮，点开会展开一个小面板，显示各桥接后台进程当前的运行状态：

- **Bridge**：MATLAB 桥接进程的状态，文案为英文原文（Running / Error / Stopped / Unavailable），不随界面语言切换。
- **Ansys**：Ansys 桥接进程的状态，文案跟随界面语言（中文界面下显示：运行中 / 已停止 / 错误 / 不可用）。
- **STK 没有对应的徽章**——STK 的连接状态只能在设置页面的 STK 卡片里查看。

## 常见连接问题

以下问题目前能在代码和设置界面里找到依据，遇到其他情况建议直接把「测试连接」返回的错误信息发给 Agent 帮你诊断。

- **MATLAB 卡片显示"未检测到"**：还没有选择 MATLAB 安装目录，去"选择 MATLAB 文件夹…"里重新选一次。
- **提示"无效的 MATLAB 路径"**：选中的文件夹不是一个有效的 MATLAB 安装目录，检查一下路径是否选对了。
- **MATLAB 换了 Python 解释器后好像没生效**：这是预期行为——MATLAB 更换解释器必须**重启应用**才会生效。
- **STK / Ansys 换了「Python 解释器（手动指定）」后好像没生效**：这也是预期行为，但触发方式不同——手动指定的解释器会在该引擎**下次启动或重建环境时**生效；想立即生效，展开对应卡片直接点「重建隔离环境」即可，不用重启整个应用。
- **引擎卡片显示"通用基座 Python 不兼容…已回退自动发现"是什么意思**：说明你在通用「Python 环境」卡片里指定的基座 Python 版本，不在这个引擎的支持范围内，该引擎已经自动回退到自己的自动发现逻辑，不影响正常使用。如果想让这个引擎也统一用你指定的基座，换一个各引擎都兼容的版本即可（比如 3.11）。
- **卡片显示"出错"**：下方会附带具体的错误说明；可以直接点"重试"按钮让 3Studio 重新准备一次。
- **"测试连接"按钮点了但显示失败**：说明桥接程序当前连不上，先确认对应软件本身能正常打开，再确认路径 / 解释器配置无误后重新测试。
- **STK 卡片提示"使用系统 Python（未隔离）"**：说明当前用的解释器不是 STK 专属的隔离环境，展开卡片点"重建隔离环境"让 3Studio 重新搭建一个独立环境（这个按钮平时也一直在，不是只有出现这条提示才有）。

> ⚠️ 待核对：桥接进程处于 Error / Stopped 状态时，除了"重试"按钮外是否还有其他官方推荐的恢复步骤（例如是否需要重启应用），当前代码里没有找到明确说明，建议后续与研发确认后补充。
